Password Manager

Description

It looks like skat finally remembered to use his password manager! One small problem though, he forgot his password to the password manager!

Can you help him log back in so he can get back to his favorite RF forums?

Analyze

가장 먼저 문제 소스코드의 주석을 보고 Path traverse 문제임을 알 수 있다.

간단한 필터링이 존재하나, …/./ 와 같이 입력하여 쉽게 우회할 수 있다.

사용자 정보를 json 파일로 저장하고, 파일을 읽어와 초기화한다.

path traverse로 user.json 파일을 읽어 credential을 획득하고, 로그인하여 해결하였다.


---

Political

Description

My new enterprise policy ensures you will remain flag-free.

Analyze

먼저 FLAG를 획득하기 위해선, valid_tokens[token]이 True인 토큰을 포함하여 redeem 엔드포인트에 접근해야 한다.

이때, token 엔드포인트에서 valid_tokens[token]이 False인 token을 발급받을 수 있고,

giveflag 엔드포인트에 admin 쿠키와 함께 발급받은 token을 전달하면 True로 설정해준다.

/giveflag?token={token} 을 전달하면 해결할 수 있지만 작동하지 않는데, 이유는 Dockerfile을 확인해 보면 알 수 있다.
Dockerfile에서 환경을 빌드하는 시점에서 policy.json 파일의 브라우저 정책을 추가한다.

policy.json 파일을 확인해 보면, 주요 단어를 glob 패턴을 사용하여 블랙리스트 방식으로 막는다.

추가된 브라우저 정책을 우회하는 방법은 다양한 방법이 있겠으나, 나는 위와 같은 방법으로 우회하여 해결하였다.
