Bandit Level 11

Level Goal

The password for the next level is stored in the file data.txt, where all lowercase (a-z) and uppercase (A-Z) letters have been rotated by 13 positions

Analyze

접속하면 data.txt 파일을 볼 수 있다.

Level Goal을 보면, 소문자부터 대문자까지 각 문자를 13자리 이동시켜놨다고 한다.

다음과 같이 tr 명령을 통해 복호화된 Password를 확인할 수 있다.

Bandit Level 12

Level Goal

The password for the next level is stored in the file data.txt, which is a hexdump of a file that has been repeatedly compressed. For this level it may be useful to create a directory under /tmp in which you can work. Use mkdir with a hard to guess directory name. Or better, use the command “mktemp -d”. Then copy the datafile using cp, and rename it using mv (read the manpages!)

Analyze

접속 시 data.txt 파일을 확인할 수 있다.

cat을 통해 열어보면, HEX dump 내용을 볼 수 있는데,

HEX dump 내용을 풀기 위해 tmp 폴더에 임시 파일을 생성하고, data.txt 파일을 복사해줬다.

xxd 명령으로 HEX dump 내용을 바이너리 형식으로 변환하고,

파일 명령어를 확인하니, gzip 파일임을 알 수 있었다.

gz 파일로 확장자를 바꾸어주고, gunzip을 통해 압축해제한다.

이후 파일을 다시 보면, bzip2로 압축된 파일임을 알 수 있고,

동일하게 확장자를 변경해주고, bunzip2로 압축을 풀어준다.

동일하게 gzip으로 압축된 파일이었고, 계속 이런 식으로 압축해제를 반복해주었다.

한 6번 정도 동일하게 압축해제하니, 다음과 같이 Password를 확인할 수 있었다.

Password

Bandit Level 13

Level Goal

The password for the next level is stored in /etc/bandit_pass/bandit14 and can only be read by user bandit14. For this level, you don’t get the next password, but you get a private SSH key that can be used to log into the next level. Note: localhost is a hostname that refers to the machine you are working on

Analyze

접속하면 개인 SSH 키를 확인할 수 있다.

해당 SSH 키를 활용하여 bandit14로 ssh 접속을 하고, /etc/bandit_pass/bandit14 파일을 읽는 것으로 보인다.

다음 명령어를 통해 접속해주고,

/etc/bandit_pass/bandit14 파일을 확인하여 Password를 읽을 수 있다.

Password

Bandit Level 14

Level Goal

The password for the next level can be retrieved by submitting the password of the current level to port 30000 on localhost.

Analyze

접속하면, 별 다른 특이한 파일은 없다.

nc 명령어로 30000 포트에 비밀번호를 보내어 다음 Password를 확인하였다.

Bandit Level 15

Level Goal

The password for the next level can be retrieved by submitting the password of the current level to port 30001 on localhost using SSL/TLS encryption.

Helpful note: Getting “DONE”, “RENEGOTIATING” or “KEYUPDATE”? Read the “CONNECTED COMMANDS” section in the manpage.

Analyze

접속하면 .bandit14.password 파일을 볼 수 있다.

openssl 명령어를 통해 SSL/TLS 암호화를 사용하여 localhost의 30001로 비밀번호를 전송할 수 있다.
이를 통해 Password 확인

Bandit Level 16

Level Goal

The credentials for the next level can be retrieved by submitting the password of the current level to a port on localhost in the range 31000 to 32000. First find out which of these ports have a server listening on them. Then find out which of those speak SSL/TLS and which don’t. There is only 1 server that will give the next credentials, the others will simply send back to you whatever you send to it.

Helpful note: Getting “DONE”, “RENEGOTIATING” or “KEYUPDATE”? Read the “CONNECTED COMMANDS” section in the manpage.

Analyze

접속하면 .bandit15.password 파일을 확인할 수 있다.

nmap을 쓰면 다음과 같이 열려 있는 포트를 볼 수 있는데,

여기서, 31790 포트에 openssl 명령어를 사용하여 개인키를 받을 수 있다.

tmp 폴더에 개인키 파일을 하나 만들고, 받은 개인키를 작성해주었다.

그리고 개인키를 통해 ssh를 연결하여 /etc/bandit_pass/bandit17 파일을 읽어 Password를 획득

Bandit Level 17

Level Goal

There are 2 files in the homedirectory: passwords.old and passwords.new. The password for the next level is in passwords.new and is the only line that has been changed between passwords.old and passwords.new

NOTE: if you have solved this level and see ‘Byebye!’ when trying to log into bandit18, this is related to the next level, bandit19

Analyze

접속하면 passwords 파일을 확인할 수 있다.

diff 명령어를 통해 다음 18레벨의 Password를 확인할 수 있다.

Bandit Level 18

Level Goal

The password for the next level is stored in a file readme in the homedirectory. Unfortunately, someone has modified .bashrc to log you out when you log in with SSH.

Analyze

계속 bandit18에 접속하니 계속 접속을 강제로 종료한다.
bandit17 레벨에서 ssh로 접속하면, 개인키 오류가 계속 떴고,

그래서, 본 컴의 cmd 창을 통해 ssh로 접속하고, 명령어를 함께 주어 Password를 확인할 수 있다.

Bandit Level 19

Level Goal

To gain access to the next level, you should use the setuid binary in the homedirectory. Execute it without arguments to find out how to use it. The password for this level can be found in the usual place (/etc/bandit_pass), after you have used the setuid binary.

Analyze

setuid가 걸려 있는 bandit20-do 파일을 확인할 수 있다.

실행을 하면 용법을 알려주는데, setuid가 걸려있기 때문에 해당 바이너리로 bandit20 권한으로 커멘드 입력이 가능하다.

위와 같이 명령어를 보내 Password를 확인 !

Password

Bandit Level 20

Level Goal

There is a setuid binary in the homedirectory that does the following: it makes a connection to localhost on the port you specify as a commandline argument. It then reads a line of text from the connection and compares it to the password in the previous level (bandit20). If the password is correct, it will transmit the password for the next level (bandit21).

NOTE: Try connecting to your own network daemon to see if it works as you think

Analyze

접속하면 setuid가 걸려 있는 suconnect 파일이 존재한다.

이게 먼 소린고 하니, nc 명령으로 두개의 bandit20 유저를 연결하고, bandit20 레벨의 Password를 보내 해결할 수 있었다.

Password

A program is running automatically at regular intervals from cron, the time-based job scheduler. Look in /etc/cron.d/ for the configuration and see what command is being executed.

./cronjob_bandit22.sh 파일을 실행하니, /tmp/t7O6lds9S0RqQh9aMcz6ShpAoZKF7fgv 파일에 대한 권한 오류를 확인할 수 있는데, 해당 파일을 확인하니 다음 Password를 확인할 수 있었다.

Bandit Level 22

Level Goal

A program is running automatically at regular intervals from cron, the time-based job scheduler. Look in /etc/cron.d/ for the configuration and see what command is being executed.

NOTE: Looking at shell scripts written by other people is a very useful skill. The script for this level is intentionally made easy to read. If you are having problems understanding what it does, try executing it to see the debug information it prints.

Analyze

접속하면, 다른 특이한 파일은 없다.
