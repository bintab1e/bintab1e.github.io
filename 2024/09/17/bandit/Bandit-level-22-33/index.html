Bandit : level 22 ~ 33

bintab1e
2024-09-17
pwnable
bandit

Bandit Level 23

Level Goal

A program is running automatically at regular intervals from cron, the time-based job scheduler. Look in /etc/cron.d/ for the configuration and see what command is being executed.

NOTE: This level requires you to create your own first shell-script. This is a very big step and you should be proud of yourself when you beat this level!

NOTE 2: Keep in mind that your shell script is removed once executed, so you may want to keep a copy around…

Analyze

접속하면 홈 디렉토리에 별 다른 중요한 파일은 보이지 않는다.

/etc/cron.d/cronjob_bandit24 파일을 읽어보니 /usr/bin/cronjob_bandit24.sh 파일을 reboot 마다 실행한다.

실행하니까, 뭐 rm 명령의 권한 오류가 뜨는데,

파일 코드를 확인해 보면, /var/spool/$myname/foo:의 모든 스크립트를 삭제한다.
이때, 사용자가 bandit23일 경우 강제 종료한다.

문제는, foo 디렉토리는 권한이 없어 읽어올 수 없었다.

다음과 같이 /tmp/exploit 디렉토리에 password.sh를 생성한다.

이후 디렉토리와 파일에 권한을 부여해주고, foo 디렉토리에 옮겨 주었다.

기다리다 보면 foo 디렉토리에서 password.sh가 cron에 의해 bandit24 권한으로 실행되어 Password를 읽어올 수 있다.

Password

Bandit Level 24

Level Goal

A daemon is listening on port 30002 and will give you the password for bandit25 if given the password for bandit24 and a secret numeric 4-digit pincode. There is no way to retrieve the pincode except by going through all of the 10000 combinations, called brute-forcing.
You do not need to create new connections each time

Analyze

접속 시 별 다른 파일은 없다.

다음과 같이 sh 코드를 짜주고

다음과 같은 password25.txt 파일을 생성해준다.

다음과 같이 같이 전달하여 Password를 확인할 수 있다.

Password

Bandit Level 25

Level Goal

Logging in to bandit26 from bandit25 should be fairly easy… The shell for user bandit26 is not /bin/bash, but something else. Find out what it is, how it works and how to break out of it.

NOTE: if you’re a Windows user and typically use Powershell to ssh into bandit: Powershell is known to cause issues with the intended solution to this level. You should use command prompt instead.

Analyze

접근하면 bandit26.sshkey 파일을 확인할 수 있다.

passwd 파일을 확인하여 bandit26의 셸을 확인할 수 있고,

다음 명령어를 통해 banit26에 접근할 수 있는데, 바로 접속이 끊어진다.

bandit26의 셸 파일을 읽어 보니, 코드가 다음 같았는데, 여기서 more 명령어는 한 페이지씩 읽어 온다.

진짜 도저히 모르겠어서, 찾아봤는데 창을 작게하고 SSH 접근을 하면 v를 입력해 vi에 들어가는데,
이때 /bandit26 Password 파일을 읽고 나오면 Password를 확인할 수 있다.

다음과 같이 입력하여 bandit26의 셸을 딸 수 있다.

Password

Analyze

홈 디렉토리에 다른 특이한 파일은 없었다.

git clone 명령을 통해 bandit29의 repo를 클론해 올 수 있다.

repo의 README.md 파일을 읽어보면 다음과 같이 적혀 있다.

git log를 읽어도 다른 Password는 확인할 수 없었고,

branch를 확인하니, 다른 branch들을 볼 수 있었다.

remotes/origin/dev의 branch로 변경하고 log를 확인하니 4개의 commit을 볼 수 있는데,

첫 번째 commit에서 Password를 확인할 수 있었다.

Level Goal

Clone the repository and find the password for the next level.

Analyze

홈 디렉토리는 다른 파일은 없었고,

git clone을 통해 bandit30의 git repo를 클론해왔다.

README 파일을 보면 다음과 같이 적혀 있어 다른 힌트를 받을 수 없었다.

branch를 확인해 보니, 다른 master branch를 제외한 다른 branch는 없었다.

git show-ref 명령을 통해 모든 참조를 확인해 보면, master 외의 다른 tag 참조를 볼 수 있었다.

해당 tag 참조의 commit을 확인하여 Password를 확인할 수 있다.
