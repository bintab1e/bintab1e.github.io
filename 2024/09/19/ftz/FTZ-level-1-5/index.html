bintab1e
2024-09-19
pwnable
FTZ

Level 1

Description

level2 권한에 setuid가 걸린 파일을 찾는다.

Analyze

위 명령을 통해 전체 파일 리스트에서 user가 level2인 setuid가 걸린 파일을 탐색할 수 있다.

결과로 다음과 같이 ExecuteMe 파일을 확인할 수 있는데, 실행하면 레벨 2의 권한으로 원하는 명령어를 실행시켜준다고 한다.

/bin/bash 명령을 실행하여 level2의 쉘을 획득할 수 있다.

Analyze

실행하면 VIM으로 들어가지는데, 다음과 같이 VIM 내부에서 /bin/bash를 입력하여 level3의 쉘을 획득할 수 있다.

Level 3

Description

다음 코드는 autodig의 소스이다.

이를 이용하여 level4의 권한을 얻어라.

more hints.

  • 동시에 여러 명령어를 사용하려면?
  • 문자열 형태로 명령어를 전달하려면?

Analyze

find 명령어를 통해 level4 권한의 setuid가 걸린 파일을 살펴 보면, autodig 파일을 찾을 수 있다.

autodig 프로그램은 받은 받은 인자로 위와 같은 명령을 수행해준다.
argv[1]에 악의적인 명령어를 입력하여 공격자의 원하는 명령을 실행시킬 수 있다.

이상하게 /bin/bash는 실행이 안되어 my-pass 명령으로 다음 패스워드를 확인할 수 있었다.

Level 4

Description

누군가 /etc/xinetd.d에 백도어를 심어놓았다.!

Analyze

xinetd.d 디렉토리로 가 보면, 다음과 같이 backdoor 파일을 확인할 수 있다.

xinetd

데몬이란, 백그라운드에서 기본적으로 실행되는 프로그램을 말한다.
일반적으론 특정 작업을 수행하기 위해 지속적으로 실행이 되는데, Xinetd는 이런 데몬을 제어하고 관리하기 위한 슈퍼 데몬이다.

이때 xinetd는 /etc/xinetd.d/ 디렉토리에 슈퍼 데몬이 관리해야 할 서비스에 대한 설정 파일을 둔다. 각 설정 파일은 서비스의 활성화 여부, 실행파일 경로, 사용 권한 등의 다양한 설정을 포함한다.

xinetd.d 디렉토리에 위치한 backdoor 파일을 읽어 보면, Xinetd 슈퍼데몬이 관리하는 finger 서비스의 설정파일로 보인다.

여기서 server 인자의 실행 프로그램을 확인해 보면 다음과 같이 my-pass 명령어를 실행하는데, level5 사용자의 권한으로 데몬이 서비스되므로, level5의 패스워드를 뱉어낼 것이다.

우선, backdoor.c 코드를 실행파일로 만들어 주기 위해 gcc로 컴파일을 수행한다.

이후 finger 서비스를 localhost 호스트로 실행하면 Level5의 패스워드를 확인할 수 있다.

Level 5

Description

/usr/bin/level5 프로그램은 /tmp 디렉토리에
level5.tmp라는 이름의 임시파일을 생성한다.

이를 이용하여 level6의 권한을 얻어라

Analyze

/usr/bin/level5를 실행해도 생성된 임시파일을 확인할 수 없다. 추측하자면, 생성하자마자 바로 지우는 것으로 보인다.

/usr/bin/level5이 /tmp 디렉토리에 level5.tmp라는 임시파일을 생성한다고 했으므로, 실행하기 전 level5.tmp 파일을 tmp 디렉토리에 생성하고,

/usr/bin/level5를 실행시키면 기존에 생성한 level5.tmp 파일에 생성할 임시파일 내용이 덮어씌워져 지워지지 않고 패스워드를 확인할 수 있다.`
