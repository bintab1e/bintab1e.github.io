FTZ : level 11

bintab1e
2024-09-20
pwnable
ftz

Level 11

Description

Analyze

attackme 프로그램의 소스코드로 보이는 코드를 보면, strcpy를 통해 인자를 str에 복사하는 것을 볼 수 있다.

strcpy는 입력 크기를 제한하지 않으므로 bof가 가능하다.

gdb를 통해 main을 까 보면, main+41 부분에 ebp-264 만큼 입력을 받고 스택에 넣고 있는 것을 볼 수 있다. str이 256바이트 인데 264 만큼을 할당한 것을 보면, 8바이트만큼의 dummy 값이 함께 들어간 것을 알 수 있다.

main의 스택 프레임 구조는 다음과 같은데, str 변수 영역을 넘는 데이터를 입력하여 ret을 삽입한 쉘 코드의 주소로 덮어 해결할 수 있다.

쉘코드 모음에서 25바이트 쉘코드를 가져오고

다음과 같이 앞 dummy(243) + shellcode(25) + Return address(4) 이렇게 채워준다.

여기서 Return address는 str 메모리 주소를 찾아서 넣어주면 된다.

gdb를 통해 main에 BP를 걸고 a를 264 만큼 입력해 실행해준다.

x/80x $esp로 스택 메모리를 확인해 보면, 0xbffffb1c + 15 정도에 str이 위치한 것을 볼 수 있다.

다음과 같이 NOP으로 243바이트를 채워주고, 25바이트의 쉘코드와 str 문자열의 시작 위치인 bffffb2b를 넣어주면, Return address가 POP되면서 str로 돌아가고, NOP으로 명령이 계속 넘어가다 쉘코드가 실행된다.

그렇게 얻은 쉘을 통해 패스워드를 확인할 수 있다.
