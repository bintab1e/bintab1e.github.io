Dreamhack - uaf_overwrite[2]

bintab1e
2024-11-06
pwnable
Dreamhack

Description

Exploit Tech: Use After Free에서 실습하는 문제입니다.

Analyze

문제 소스는 다음과 같다.

바이너리에 모든 보호기법이 걸려 있다.

Human 구조체와 Robot 구조체는 같은 크기의 구조체이다. 이때, 각각의 human_func 함수와 robot_func 함수에서 메모리 영역 할당 시 할당한 메모리 영역을 초기화하지 않는다.

Human 구조체와 Robot 구조체는 동일한 영역을 할당받고, 이를 통해 Use After Free 트리거가 가능하다. Human의 age와 Robot의 fptr 위치가 동일하므로, human_func 호출 시 age에 one gadget을 입력하고, robot_func를 호출하여 fptr에 남아 있는 one gadget을 호출할 수 있다.

custom_func 함수 또한, 0x100 이상의 크기의 청크를 할당하고 해제할 수 있지만, 메모리 영역을 초기화하지 않아 Use After Free 트리거가 가능하다.

unsorted bin에 libc 영역 주소와 연결 리스트를 형성하는 것을 이용하여 UAF 취약점으로 fd나 bk 값을 읽어 libc base를 구할 수 있다.

custom_func 함수가 0x100 이상의 청크를 할당하고 해제할 수 있으므로, 해당 함수를 통해 0x410보다 큰 크기의 청크를 할당하고 해제하여 unsorted bin에 연결하고, 재할당하여 libc base를 구한다.

구한 0x7fff7fa5ce0 주소를 보면 libc 영역에 속한 것을 볼 수 있다.
