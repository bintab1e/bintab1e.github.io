CTF-writeup

Passwordless

Description

Tired of storing passwords? No worries! This super secure website is passwordless!

Analyze

app.py

/ 경로에서 POST로 username을 받아 username와 leet를 통해 uid를 생성하고 있다.

생성된 uid 경로로 접근해서 username이 admin123로 생성된 uid 값일 경우 flag를 보여준다.

취약점은 다음과 같이 leet 값을 생성하는 값이 하드코딩 되어 있어 하드코딩된 값과 admin123을 통해 admin123의 uid 값을 생성할 수 있다.

Exploit

다음과 같이 입력하여 admin123의 uid를 계산하고 uid 경로로 접근하면 flag를 확인할 수 있다.

Focus On YourSELF

Description

Have you focused on yourself recently?

Analyze

docker-compose.yaml

문제에서 가장 처음 다음 docker-compose 파일을 제공한다. 확인할 부분은 환경변수에 FLAG가 담겨 있다는 것을 알 수 있다.

/view

view 경로로 접근하면 다음과 같이 이미지를 image 파라미터를 통해 호출하는 것을 볼 수 있는데, 간단하게 LFI가 터진다.

Exploit

/proc/self/environ 해당 경로에서 현재 환경변수를 확인할 수 있으며, FLAG도 담겨 있는 것을 볼 수 있다.

File Sharing Portal

Description

Welcome to the file sharing portal! We only support tar files!

Analyze

간단하게 압축된 tar 파일을 올리면 tar 파일을 압축해제하고 담겨 있는 파일 list를 보여준다.

다음과 같이 /app/requirements.txt에 심볼릭 링크를 걸고 tar 파일로 압축하여 업로드할 경우 압축해제된 app 파일에 접근하면 심볼릭 링크가 걸려 있는 requirements.txt 파일을 볼 수 있다.

이렇게 해서 FLAG 파일을 읽어 오면 되는데, 문제는 다음과 같이 FLAG 파일의 이름을 몰라 심볼릭 링크를 걸 수 없다.

여기서 조금 시간이 걸렸는데, RCE가 필요하다 생각했지만, RCE 없이 우리가 알고 있는 방법으로 FLAG 파일을 읽어올 수 있다.
