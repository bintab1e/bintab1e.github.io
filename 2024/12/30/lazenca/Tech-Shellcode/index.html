일반적으로 어셈블리어로 작성 후 기계어로 변환하는 과정을 거쳐 생성한다.

Linux system call in assembly

C언어는 다양한 운영체제와 아키텍처에서 호환될 수 있도록 표준 라이브러리를 제공한다.
표준 라이브러리는 내부적으로 System call을 적절히 호출해 다양한 시스템에서 컴파일과 실행을 가능하게 한다.

Assembly 언어는 특정 아키텍처에 종속적인 특징을 가지고 있다.
표준 라이브러리에대한 호환성이 없기 때문에, 시스템을 제어하기 위한 System call을 직접 호출해야 한다.
이때, System call을 호출할 때는 지정된 System call 번호를 사용하여 호출한다.

System call 번호는 해당하는 시스템의 운영체제와 프로세서 아키텍처에 따라 다르다.

Change to Shellcode format

Shellcode는 운영체제나 라이브러리의 도움 없이 독립적으로 실행되어야 한다.

Assembly 코드는 링커를 통해 외부 라이브러리 및 심볼과 연결된다. 그러나 Shellcode는 이런 링킹 과정 없이 독립적으로 실행되야 하므로 텍스트 및 데이터 세그먼트를 사용할 수 없다.

따라서, 위에 작성된 예시는 Shellcode라고 할 수 없으며, 텍스트 세그먼트나 데이터 세그먼트에 의존하지 않고 메모리 내에서 모든 작업을 수행해야 한다.

write 시스템 콜을 통해 문자열을 출력하고자 하면, 두 번째 인자로, 출력하고자 하는 문자열의 주소를 전달해야 한다. 하지만, 데이터 세그먼트를 사용할 수 없으므로, mov 명령어를 통해 문자열의 주소를 직접 레지스터에 로드할 수 없다.

이는, call과 ret 명령어를 사용해 해결한다.

  1. call 명령어로 함수를 호출
  2. call 다음 출력하고자 하는 문자열을 배치
  3. call 명령어가 실행되면, 다음 리턴 주소인 문자열의 주소를 스택에 저장
  4. pop 명령어를 사용하여 스택에 저장된 리턴 주소를 ecx 레지스터에 저장
  5. write 시스템 콜의 인자로 ecx 레지스터에 전달

Debugging

strcpy() 함수가 호출되는 곳에 BP를 설고 실행해 보면, strcpy() 함수가 호출되기 이전이기 떄문에, code 변수(0x804a074)에는 아무런 값도 저장되어 있지 않다.

이때, shellcode 변수 영역에 작성한 shellcode가 저장되어 있는 것을 볼 수 있다.

에러 발생의 이유는, shellcode에 담겨 있는 null byte(\x00)가 문자열의 끝으로 인식되기 때문이다.

strlen() 함수와 strcpy() 함수는 null byte 이전까지만 데이터를 계산하고 복사한다.

이 때문에, Shellcode가 code 영역에 전체가 복사되지 않아 실행시 에러가 발생한다.

Remove null byte

아래 덤프된 코드 보면, 각각의 null byte를 확인할 수 있다.

Call instruction

call 명령어의 피연산자는 32 비트(dword)를 사용하고 있어, 작은 값(0x14)이 할당될 경우, 남은 부분에 null byte가 포함된다.

이를 해결하기 위해, jmp 명령어를 사용하여 helloworld 함수를 건너뛰고, last 함수로 이동할 수 있다.

last 함수에서 helloworld 함수를 호출하고, 다음 출력할 메세지를 저장하여 null byte를 제거할 수 있다.

ASM32-2.s

call 명령어에서 null byte가 제거된 이유는 음수 값 때문이다.

jmp short 명령어는 -128 ~ 127 범위 값을 사용하여 0x20 바이트만큼 이동한다.

그러나, call 명령어는 helloworld 함수를 호출하기 위해, -0x22로 이동해야 한다. 이때, 음수 값을 표현하기 위해 2의 보수로 변환되어 0xFFFFFFDD로 표현되어 null byte를 제거할 수 있다.

Register

call 명령어 다음의 null byte가 포함된 이유는 명령어 피연산자 값이 32 비트로 확장되면서 발생한다.

64비트, 32비트, 16비트 Register에 값이 저장될 때, 표현 가능한 최대 값보다 작은 값이 저장되어 남는 공간이 null byte로 채워진다. 이런 특성 때문에 Register에 값을 저장할 때 null byte가 발생한다.

이는 xor 명령어를 통해 해결할 수 있다.

xor 명령어는 두 피연산자 값을 배타적 논리합으로 초기화하여, 결과적으로 항상 0을 만든다.

이후, bl, cl, dl Register에 값을 저장하여 null byte를 제거할수 있다.
